# Задание

Для выполнения проекта необходимо разработать веб-приложение для проведения квизов, которое будет включать в себя следующие функции:

- Регистрация и авторизация пользователей — участников и организаторов
- Создание и настройка квизов: выбор категорий вопросов, настройка времени и правил проведения
- Возможность добавления организатором вопросов разных типов: текстовых и в виде изображения, содиночным и множественным выбором ответа
- Функция запуска опроса с возможностью подключения участников к текущему квизу по коду комнаты. Каждое задание квиза отображается в режиме реального времени для всех участников и доступно для ответа только во время его демонстрации
- Система подсчёта баллов и определения победителей
- Личный кабинет для участников и организаторов с историей участия и проведённых квизов

## Алгоритм выполнения проекта

1. Проектирование системы пользовательского интерфейса в Miro и Figma
2. Разработка модели базы данных
3. Разработка веб-интерфейса на основе HTML, CSS, JavaScript и одного из популярных фреймворков
4. Создание серверной части на JavaScript для взаимодействия с базой данных
5. Настройка работы приложения с WebSocket
6. Тестирование разработанной функциональности на группе пользователей

## Какие навыки и компетенции нужны для выполнения

- Уверенное знание HTML, CSS, JavaScript
- Навыки работы с React, Next.js или любым другим фреймворком
- Умение создавать серверную часть на Node.js или любом другом фреймворке
- Навыки работы в Miro и Figma
- Будет плюсом опыт работы с WebSoсket (Soket.IO, PartySocket)

## Какой результат хотим получить

Работоспособный прототип (MVP), позволяющий:

- создавать опрос в формате квиза (организатору)
- настраивать и запускать квиз
- подключаться к активному квизу для прохождения опроса в реальном времени (участникам)
- показывать результат всем пользователям в виде лидерборда по окончании квиза и сохранять данные

## В каком виде загрузить результат на платформу

Презентация или краткая пояснительная записка (до 5 страниц) с обоснованием выбора средств разработки и описанием основных этапов разработки. В приложении обязательны: ссылка на макеты, ссылка на репозиторий с кодовой базой, ссылка на работоспособный продукт (опционально).

## Критерии выбора лучших работ

- Соответствие функциональности заявленным требованиям
- Удобство и простота использования
- Стабильность работы без сбоев и ошибок клиентской и серверной частей веб-приложения
- Креативность в дизайне и функциональности приветствуется
